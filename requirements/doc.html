Actividad Colaborativa II

Taller Integrador – Frontend SPA del Supermercado (React)

Objetivo de aprendizaje

● Desarrollar una Single Page Application (SPA) utilizando React.

● Consumir una API REST mediante Axios.

● Implementar interfaces CRUD para múltiples módulos del sistema.

● Diseñar interfaces responsivas mediante Bootstrap.

Contexto

Luego de implementar el backend del sistema de supermercado, la empresa MarketSoft necesita desarrollar la interfaz web que permita a los usuarios interactuar con el sistema.

La aplicación será una SPA (Single Page Application) que consumirá la API desarrollada en la actividad anterior.

La aplicación permitirá gestionar los módulos:

  • Productos
  • Usuarios
  • Proveedores
  • Ventas

Cada módulo permitirá visualizar, crear, actualizar y eliminar registros.

Lineamientos técnicos obligatorios

  • Framework: React
  • Consumo de API: Axios
  • Estilos: Bootstrap

Navegación

Implementar rutas mediante:

react-router-dom

Consumo de API

Todas las operaciones CRUD deben consumir los endpoints del backend.

Ejemplo:

GET /api/products

POST /api/products

PUT /api/products/:id

DELETE /api/products/:id

Formularios

Cada módulo debe tener:

  • formulario de creación
  • formulario de edición

Tablas

Mostrar los datos en tablas con:

  • editar
  • eliminar
  • crear

Repositorio

Repositorio Git público obligatorio.

La aplicación debe ejecutarse con:

npm install
npm start

Penalizaciones

Lógica de negocio en componentes incorrectos.

  • No consumir la API real.
  • Código duplicado.
  • Proyecto que no compila.

Entrega

  • Trabajo en equipo: máximo 4 integrantes.
  • Repositorio GitHub público obligatorio.
  • El README debe incluir:
    • nombres de integrantes
    • instrucciones de ejecución
    • descripción de la arquitectura
  • El proyecto debe compilar sin errores.
